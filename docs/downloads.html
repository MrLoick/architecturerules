Assert Your Architecture! with this open source java library. Architecture Rules leverages an xml configuration file and optional programatic configuration to assert your code's architecture via unit tests or ant tasks. This test is able to assert that specific packages do not depend on others and is able to check for and report on cyclic dependencies among your project's packages and classes. This project wraps a industry accepted JDepend to simplify the process of maintaining a solid software architecture.

Release 2.0-rc1

enhancement release

First real release. This probobly should have been the 1.0 series. This release finished the issues and tasks that should have been completed before attempting to release the tool. It added a real xml parsing strategy to the ConfigurationFactory - commons digester. Also, test coverage is now at 88%, beating the goal of 80%. And finally, rather than just reporting that a cyclic dependency exists, the exact packages with the cyclic dependency are exposed. Soon, even the classes will be identified.
